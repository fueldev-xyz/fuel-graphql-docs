# 枚举

## `ConsensusParametersVersion`

共识参数的版本。

`V1`：
版本 1。

## `ContractParametersVersion`

合同特定共识参数的版本。

`V1`：
版本 1。

## `FeeParametersVersion`

费用特定共识参数的版本。

`V1`：
版本 1。

## `GasCostsVersion`

燃气特定共识参数的版本。

`V1`：
版本 1。

## `MessageState`

消息的状态，可以是 `UNSPENT`，`SPENT` 或 `NOT_FOUND`。

`UNSPENT`：
消息未被花费。

`SPENT`：
消息已被花费。

`NOT_FOUND`：
未找到消息。

## `ReceiptType`

指示生成收据的交易类型的收据类型。

`CALL`：
收据由合同调用生成。

`RETURN`：
收据由没有返回数据的交易生成。

`RETURN_DATA`：
收据由返回数据的交易生成。

`PANIC`：
收据由合同调用失败时生成。

`REVERT`：
收据由合同调用回滚时生成。

`LOG`：
收据由合同中的日志生成。日志收据是为非引用类型生成的，即 `bool`，`u8`，`u16`，`u32` 和 `u64`。

`LOG_DATA`：
收据由合同中的日志生成。日志数据是为引用类型生成的，引用类型包括除上述非引用类型之外的所有类型。

`TRANSFER`：
收据由向合同转移硬币的交易生成。

`TRANSFER_OUT`：
收据由向地址（而不是合同）转移硬币的交易生成。

`SCRIPT_RESULT`：
收据由脚本生成。

`MESSAGE_OUT`：
收据由消息生成。

`MINT`：
收据由铸币生成。

`BURN`：
收据由烧毁生成。

## `PredicateParametersVersion`

谓词特定共识参数的版本。

`V1`：
版本 1。

## `ReturnType`

交易返回响应的类型。

`RETURN`：
表示交易没有返回任何数据。

`RETURN_DATA`：
表示交易返回了一些数据。

`REVERT`：
表示交易回滚。

## `RunState`

[`RunResult`](/docs/reference/objects/#runresult) 的状态。

`COMPLETED`：
所有断点已处理，程序已终止。

`BREAKPOINT`：
程序在断点处停止。

## `ScriptParametersVersion`

脚本特定共识参数的版本。

`V1`：
版本 1。

## `TxParametersVersion`

交易特定共识参数的版本。

`V1`：
版本 1。
