# 订阅

## `statusChange`

返回给定交易ID的[`TransactionStatus!`](/docs/reference/unions/#transactionstatus)更新流，如果当前状态是`[TransactionStatus::Submitted]`。

这个流会永远等待，所以建议在超时内使用。如果流的轮询速度低于更新到达的速度，则可能会错过更新。在这种情况下，流将关闭而没有状态。如果发生这种情况，可以简单地重新启动流以返回最新状态。

**参数:**

`id`: [`TransactionId!`](/docs/reference/scalars/#transactionid)

要流式传输状态更新的交易ID。

## `submitAndAwait`

将交易提交到`TxPool`并等待返回[`TransactionStatus!`](/docs/reference/unions/#transactionstatus)。

**参数:**

`tx`: [`HexString!`](/docs/reference/scalars/#hexstring)

交易的十六进制字符串。
